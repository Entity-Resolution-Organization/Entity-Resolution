## Design Context

### Users
Mixed technical audience at a Google HQ demo: Google engineers, PMs, and academic evaluators (course instructors, TAs). They will evaluate both visual polish and technical depth. The interface must impress at first glance while rewarding closer inspection with well-structured data flows and production-readiness signals.

### Brand Personality
**Precise. Confident. Clean.**

Enterprise SaaS tone — like Palantir, Quantexa, or Tilores. The interface communicates through clarity and restraint, not decoration. Data is the hero. Every element earns its space. Trust comes from showing the system works correctly, not from visual spectacle.

Voice: Direct, technical, lowercase sentence case. No exclamation marks, no "powered by AI" language, no marketing superlatives. State what the system does in plain terms.

### Aesthetic Direction

**Visual tone:** Dark enterprise data platform. Depth through subtle layering (glass surfaces, tinted shadows), not through color variety. Information-dense where appropriate (results, metrics) but spacious where focus matters (inputs, hero).

**Theme:** Dark mode only. Deep navy-black background (#060a10) with a single indigo accent (#818cf8). Semantic colors reserved strictly for status: green (match/ok), amber (review/warn), red (no-match/error). No decorative color.

**Typography:** Outfit (Google Fonts) for all text. Tight tracking on display sizes, standard for body. Tabular nums for all data. Monospace (SF Mono) for probabilities and latency values.

**Motion:** Spring physics (Framer Motion) for state transitions. Staggered reveals for lists. No entrance animations on page load beyond a single fade. Motion should feel weighted and physical, never bouncy or playful.

**Surfaces:** True glassmorphism — backdrop blur + inner highlight border + tinted shadow. Grain overlay at low opacity for texture. No flat cards.

**Reference products:** Palantir Foundry, Quantexa, Linear, Vercel Dashboard.

**Anti-references:** Generic AI demos (purple/blue rainbow gradients, chatbot UIs, "Powered by AI" badges). Academic prototypes (unstyled defaults, Bootstrap look). Consumer SaaS (playful illustrations, bright candy colors, rounded friendly shapes). This should feel like a shipped product, not a hackathon project.

### Design Principles

1. **Data is the interface.** Match scores, attribution charts, and decision badges are the primary content. Chrome exists to organize data, not to decorate.

2. **One accent, semantic status.** Indigo is the single interactive accent. Green, amber, and red communicate match/review/no-match decisions only. No other colors.

3. **Earn every element.** If a card, border, shadow, or animation doesn't serve comprehension, remove it. Density is acceptable; clutter is not.

4. **Precision over personality.** Sentence case headings, tabular numerals, monospace for data, no emoji, no exclamation marks. The interface should read like a technical instrument panel.

5. **Accessible by default.** WCAG AA contrast ratios, visible focus indicators, keyboard navigable, reduced-motion safe. These are not optional polish — they are structural requirements.

### Technical Constraints

- **Stack:** React 19 + Vite 8 + Tailwind CSS v4 + Framer Motion
- **Icons:** Lucide React (already installed, do not switch)
- **Charts:** Plotly.js + react-force-graph-2d
- **Font:** Outfit via Google Fonts CDN (400, 500, 600, 700, 800)
- **Dark mode only** — no light mode toggle needed
- **No new dependencies** without explicit approval
